# Spring AI POC

A Spring Boot application implementing a **Retrieval Augmented Generation (RAG)** system. This system allows users to upload and parse PDF documents, ingest text into a vector database, and query documents using natural language with answers generated by a local LLM.

## Overview

The application provides a complete pipeline for:
- **Document Ingestion**: Upload PDF files or plain text
- **Vector Embedding**: Convert documents into embeddings using local models
- **Semantic Search**: Query stored documents using vector similarity
- **Answer Generation**: Generate contextual answers using a local LLM

## Architecture

### High-Level Flow

```
1. Ingestion Pipeline:
   Documents (PDF/Text) → TextChunker → EmbeddingModel → Qdrant (Vector DB)

2. Query Pipeline:
   Question → EmbeddingModel → Qdrant Search → ContextRetrieval → LLM → Answer
```

### Key Components

| Component | Purpose |
|-----------|---------|
| **ChatController** | REST endpoints for text ingestion (`/ingest`) and queries (`/ask`) |
| **PdfController** | Multipart endpoint (`/upload-pdf`) for PDF document processing |
| **DocumentService** | Handles text ingestion and PDF chunking into embeddings |
| **RagService** | Implements RAG pattern with semantic search and answer generation |
| **PdfParserService** | Extracts text from PDFs using PDFBox |
| **TextChunker** | Splits large documents into fixed-size chunks (1000 chars default) |
| **AiConfig** | Bean configuration for LLM, embeddings, and vector store |

## Prerequisites

### System Requirements
- **Java**: 21 or higher
- **Maven**: 3.6+
- **Docker & Docker Compose**: For running Qdrant
- **RAM**: Minimum 4GB (8GB+ recommended)
- **CPU**: Multi-core processor

### External Services
- **Ollama** (http://localhost:11434): Provides LLM and embedding models
- **Qdrant** (localhost:6334 gRPC, 6333 REST): Vector database

## Installation & Setup

### 1. Clone Repository
```bash
git clone <repository-url>
cd spring-ai-poc
```

### 2. Start Qdrant (Vector Database)
```bash
docker-compose up -d
```

Verify Qdrant is running:
```bash
# Access UI
open http://localhost:6333/dashboard

# Check container status
docker ps | grep qdrant
```

### 3. Install & Start Ollama

#### macOS
```bash
# Download from https://ollama.ai/download/mac
# Or install via Homebrew
brew install ollama

# Start Ollama service
ollama serve
```

#### Linux
```bash
curl -fsSL https://ollama.ai/install.sh | sh
ollama serve
```

#### Windows
Download installer from https://ollama.ai/download/windows

### 4. Pull Required Models

In a new terminal:
```bash
# Pull chat model
ollama pull phi3:mini

# Pull embedding model
ollama pull nomic-embed-text

# Verify models are installed
ollama list
```

**Model Details:**
- **Chat Model**: `phi3:mini` (lightweight, ~2.7GB) - for answer generation
- **Embedding Model**: `nomic-embed-text` (274MB) - for semantic embeddings

### 5. Build & Run Application

```bash
# Build the project
mvn clean package

# Run the application
mvn spring-boot:run
```

Application starts on `http://localhost:8080`

## API Endpoints

### Document Ingestion

#### 1. Ingest Plain Text
```bash
curl -X POST http://localhost:8080/ingest \
  -H "Content-Type: text/plain" \
  -d "Your text content here"
```

**Response**:
```

```

---

#### 2. Upload PDF Document
```bash
curl -X POST http://localhost:8080/upload-pdf \
  -F "file=@document.pdf"
```

**Response**:
```
PDF processed and stored in vector DB
```

### Query Documents

#### Ask a Question
```bash
curl "http://localhost:8080/ask?question=What%20is%20the%20main%20topic?"
```

**Response**:
```
<Generated answer based on retrieved context>
```

## Troubleshooting

### Issue: "UNAVAILABLE: io exception" (Qdrant Connection Error)
**Solution**: Ensure Qdrant is running with port 6334 exposed
```bash
docker-compose down
docker-compose up -d
docker ps | grep qdrant
```

### Issue: "Timeout" Error
**Solution**: Model is slow or not using GPU
```bash
# Check if Ollama is running
ps aux | grep ollama

# Test model directly
ollama run phi3:mini "Hello"

# If too slow, try lighter model
ollama pull tinyllama
# Then update AiConfig.java to use "tinyllama"
```

### Issue: "Model not found" Error
**Solution**: Pull the required models
```bash
ollama pull phi3:mini
ollama pull nomic-embed-text
ollama list
```

### Issue: Qdrant Collection Not Found
**Solution**: Collection is created automatically on first document ingestion. If needed, manually create via REST API:
```bash
curl -X PUT http://localhost:6333/collections/documents \
  -H "Content-Type: application/json" \
  -d '{
    "vectors": {
      "size": 768,
      "distance": "Cosine"
    }
  }'
```

## Performance Notes

- **Embedding Size**: 768 dimensions (nomic-embed-text standard)
- **Chunk Size**: 1000 characters (configurable in TextChunker)
- **Search Limit**: Top 3 results per query
- **Model Context**: Limited by Ollama model capabilities
- **GPU Support**: Enabled if available (MLX on Apple Silicon, CUDA on NVIDIA)

## Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| Connection refused to Qdrant | Port 6334 not exposed | Recreate container with port mapping |
| Model timeout | GPU not enabled or model too large | Check Ollama logs, reduce timeout, use smaller model |
| Out of memory | Large documents or model too big | Increase chunk size, reduce model size, increase system RAM |
| PDF parsing fails | Corrupted PDF or unsupported format | Try with different PDF, ensure PDFBox can read it |
